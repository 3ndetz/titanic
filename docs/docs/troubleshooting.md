# Troubleshooting

Разбор частых насущных проблем и вопросов при работе с проектом.

## DVC А если я откатился на прошлую ветку, как сихнорнизировать dvc?

Если вы откатились на прошлую ветку или коммит в Git, чтобы синхронизировать данные с DVC, выполните следующие шаги:

### 1. **Переключитесь на нужную ветку или коммит**

   Используйте Git для переключения:

   ```bash
   git checkout <branch_or_commit>
   ```

### 2. **Синхронизируйте данные с помощью DVC**

   После переключения выполните команду:

   ```bash
   dvc pull
   ```

   Эта команда загрузит данные из удаленного хранилища, соответствующие текущему состоянию .dvc файлов или dvc.lock.

### 3. **Проверьте состояние данных**

   Убедитесь, что данные восстановлены корректно:

   ```bash
   dvc status
   ```

   Если все синхронизировано, команда покажет, что данные актуальны.

### 4. **(Опционально) Восстановите данные вручную**

   Если вы хотите восстановить конкретные данные, вы можете использовать:

   ```bash
   dvc checkout
   ```

   Эта команда восстановит данные, соответствующие текущему состоянию .dvc файлов.

### Пример полного процесса

1. Переключение на ветку:

   ```bash
   git checkout main
   ```

2. Синхронизация данных:

   ```bash
   dvc pull
   ```

3. Проверка состояния:

   ```bash
   dvc status
   ```

Теперь ваш проект синхронизирован с состоянием данных, соответствующим выбранной ветке или коммиту.
